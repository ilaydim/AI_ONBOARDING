# Product Management Ekibi

Sprint yönetimi, paydaş iletişimi ve ürün stratejisinden sorumlu 4 kişilik ekip.

## Ürün Alanları

| Alan | Sorumlu PM | Takım |
|---|---|---|
| TechNova Suite (core) | Deniz Aydın | Backend + Frontend |
| Analytics Platformu | Selin Kara | Data + Backend |
| API Gateway & Entegrasyonlar | Emre Öztürk | Platform ekibi |
| Mobil Uygulamalar | Ayda Çelik | Mobile ekibi |

## Metodoloji: Dual-Track Agile

- **Discovery Track**: Kullanıcı araştırması, hipotez testi, prototipleme
- **Delivery Track**: 2 haftalık sprint, geliştirme, QA, release

## PM — Mühendislik Akışı

1. PM → PRD yazar (Confluence'da)
2. PRD'i mühendislik liderine sunar (Refinement)
3. Teknik tasarım yazılır (Backend lead)
4. Tickets Jira'ya taşınır, sprint'e alınır
5. Demo → Onay → Release

## Önceliklendirme: RICE Skoru

```
RICE = (Reach × Impact × Confidence) / Effort
```

- **Reach**: 1 ayda etkilenen kullanıcı sayısı
- **Impact**: 0.25 / 0.5 / 1 / 2 / 3 (etkisiz → dönüştürücü)
- **Confidence**: 0–100%
- **Effort**: Kişi-hafta

## OKR Döngüsü

- OKR'lar çeyrek başında belirlenir (Ocak / Nisan / Temmuz / Ekim)
- Her sprint'te key result'lar güncellenir
- Çeyrek sonunda retrospektif yapılır

## Önemli Dökümanlar

- PRD şablonu → Confluence > Product > Templates
- Roadmap → Jira > Roadmap view
- Sprint review notları → Confluence > Sprint Reviews
