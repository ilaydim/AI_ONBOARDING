# Backend Team

The team responsible for building and operating the server-side architecture of TechNova Suite.

## Team Structure
- 12 backend engineers (3 junior, 5 mid, 4 senior)
- 1 Staff Engineer (architecture decisions)
- 1 Engineering Manager

## Tech Stack
- **Language**: Python 3.11+ (FastAPI), Go (performance-critical services)
- **Database**: PostgreSQL (primary), Redis (cache), Elasticsearch (search)
- **Messaging**: Kafka
- **Container**: Docker + Kubernetes
- **CI/CD**: GitHub Actions + ArgoCD

## Responsibilities
- REST and GraphQL API development
- Data modeling and migration management
- Performance optimization
- Microservice architecture and inter-service communication
- On-call rotation for production incidents

## Architecture Overview
TechNova Suite backend follows a microservices architecture:
- **api-gateway**: Routing, auth, rate limiting
- **user-service**: Authentication, authorization, user profiles
- **product-service**: ERP product catalog
- **analytics-service**: Real-time event processing (Go)
- **notification-service**: Email, SMS, push notifications

## Key Standards
- All APIs documented with OpenAPI (auto-generated by FastAPI)
- Every endpoint requires authentication unless explicitly public
- Database migrations via Alembic — never alter tables manually
- All secrets via environment variables, never hardcoded
- Code coverage minimum 70% required to merge

## Team Norms
- PR review turnaround: within 1 business day
- On-call rotation: 1 week each, PagerDuty
- Architecture decisions documented as ADRs in Confluence
- Staff Engineer must approve changes to shared libraries
