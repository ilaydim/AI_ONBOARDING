# TechNova — Work Culture & Values

## How We Work

### Communication
- **Default to async**: We use Slack for most communication. Tag people only when urgent.
- **Meetings have agendas**: No agenda, no meeting. All meetings are documented.
- **Write things down**: Decisions, context, and rationale go into Confluence or PR descriptions.
- **English for technical docs**, Turkish for internal discussions (both are welcome).

### Code Culture
- All code goes through pull requests — minimum 1 reviewer, 2 for critical paths.
- We write tests before merging. Coverage below 70% blocks the PR.
- Code review is about the code, not the person. Feedback is constructive.
- No hero culture: if you're the only one who understands something, that's a bug.

### Meetings & Rituals
- **Daily standup**: 15 min, async-first via Slack bot (Monday–Friday)
- **Sprint planning**: Every 2 weeks, Monday morning
- **Retrospective**: End of each sprint, Fridays at 16:00
- **All-hands**: Monthly, last Thursday of the month
- **1:1s**: Weekly between engineer and manager

### Working Hours
- Core hours: 10:00–17:00 Istanbul time
- Flexible start/end outside core hours
- Remote-first: most teams are distributed across Istanbul and Ankara

### Performance & Growth
- OKRs set quarterly at team + individual level
- 360° feedback every 6 months
- Learning budget: 3,000 TL/year per person for books, courses, conferences
- Internal tech talks every other Friday ("Nova Tech Talks")

### On-Call
- Backend, DevOps, and Platform engineers rotate on-call
- 1-week rotation, PagerDuty alerts
- Incidents are followed by blameless post-mortems

## Onboarding Expectations

**Week 1**: Meet your team, set up local environment, read key architecture docs  
**Week 2–3**: Complete your learning path tasks, shadow your buddy  
**Month 1**: Merge your first PR  
**Month 3**: Own a feature end-to-end
